# genshin_auto_nav

Автономное движение персонажа по маршруту в Genshin Impact **без доступа к памяти/API игры** — только захват экрана, компьютерное зрение и симуляция ввода.

> ⚠️ Образовательный/исследовательский проект по CV и управлению. Автоматизация может нарушать ToS игры — использовать на свой риск, желательно офлайн/в одиночном контексте.

## Идея

| Слой | Что делает | Частота | Железо |
|------|-----------|---------|--------|
| 1. Миникарта | абсолютная привязка координат + курс | ~30 Гц | CPU |
| 2. SAM3 | маски объектов (уточнение позиции) | по событию | GPU/iGPU/NPU |
| 3. Семантика | смысл масок (дерево/монстр/камень) | по событию | GPU/iGPU/NPU |
| 4. Триангуляция | глубина объектов по покачиванию камеры/шагу/прыжку | по событию | CPU |
| 5. Применение | детект препятствий в коридоре движения | ~30 Гц | CPU |
| 6. Детект застревания | позиция не меняется при зажатой кнопке | ~30 Гц | CPU |
| 7. Control loop | state machine, ведёт по маршруту, обходит препятствия | ~30-60 Гц | CPU |

Между тяжёлыми SAM-кейфреймами позиции объектов тащит **CPU optical flow** — это единственный слой трекинга, который реально снимает нагрузку с дискретной GPU (а не просто делает её легче), что критично для контеншна с самой игрой.

## Структура

```
genshin_auto_nav/
├── run.py                       # точка входа
├── config.yaml                  # вся конфигурация
├── requirements.txt
├── routes/example_route.json    # маршрут (точки в метрах)
├── scripts/
│   ├── setup_venv.bat           # установка (Python 3.12, кэш pip на G:)
│   └── record_route.py          # запись маршрута руками
├── docs/architecture.md         # подробности фьюжна и математики
└── genshin_nav/
    ├── config.py                # датаклассы конфига
    ├── capture/screen_capture.py        # захват (dxcam/mss)
    ├── minimap/minimap_reader.py        # шаг 1: позиция + курс
    ├── perception/
    │   ├── optical_flow.py              # CPU-параллакс + метрика уверенности
    │   ├── keyframe.py                  # событийный триггер SAM
    │   ├── sam_segmenter.py             # шаг 2: SAM3 (torch/openvino, ROI)
    │   └── semantic.py                  # шаг 3: смысловая разметка
    ├── geometry/triangulation.py        # шаг 4-5: математика триангуляции
    ├── fusion/estimator.py              # объединение миникарты + триангуляции (Kalman)
    ├── control/
    │   ├── input_sim.py                 # симуляция клавиш/мыши (pydirectinput)
    │   ├── stuck_detector.py            # шаг 6: детект застревания
    │   └── navigator.py                 # шаг 7: control loop / state machine
    └── utils/geom.py                    # углы, пинхол-камера, повороты
```

## Установка

```bat
scripts\setup_venv.bat
```
Скрипт ставит окружение на **Python 3.12** и гонит весь pip temp/cache на `G:` (диск C: переполнен). Тяжёлые `torch`/`transformers`/SAM3-веса/OpenVINO ставятся отдельно — команды печатает сам скрипт.

## Запуск

```bat
.venv\Scripts\activate
python run.py --config config.yaml --route routes\example_route.json
```
Аварийная остановка: **F12** или **Ctrl+C**.

## Калибровка (обязательно перед первым запуском)

1. `capture.region` / `window_title` — окно игры.
2. `minimap.region` — точный прямоугольник миникарты (под своё разрешение).
3. `camera.deg_per_mouse_unit` — сколько градусов на единицу мышемува (от чувствительности в игре).
4. `camera.orbit_radius_m` — радиус орбиты камеры 3-го лица (≈4-5 м).
5. `minimap.world_atlas_path` — путь к сшитому атласу мира для глобальной локализации (иначе работа в относительных координатах от старта).
